# 外部でのモデル学習に関する注意事項

このドキュメントは、「8-bit-music-maker」で使用する自作モデルを外部（Python環境など）で学習・エクスポートする際の重要なポイントをまとめたものです。

## 1. モデル形式と変換の互換性
Magenta.js で動作させるためには、Python（TensorFlow）で作成したチェックポイントを **TensorFlow.js 形式 (.json + .bin)** に変換する必要があります。

*   **バージョンの一致**: Python 側の `magenta` ライブラリと、フロントエンドの `@magenta/music` 側のバージョンを可能な限り一致させてください。
*   **コンバーター**: `tensorflowjs_converter` もしくは Magenta 公式の `magenta_model_converter` を使用して変換します。

## 2. モデルアーキテクチャの定義
現在のアプリ構成（Trioモデル想定）に合わせる必要があります。

*   **Trioモデル**: メロディ、ベース、ドラム（パーカッション）の3トラックを同時に生成する構成です。
*   **量子化（Quantization）**: 1小節あたりのステップ数（例：16ステップ）を、学習データ（MIDI）とUI側の設定で一致させてください。
*   **ノート範囲**: 生成される楽器の音域が、アプリ側の `audioEngine.ts` で定義されている音域と合っているか確認してください。

## 3. ファイルサイズの最適化
ブラウザ上での読み込み速度を向上させるために重要です。

*   **重みの量子化**: エクスポート時に 16bit または 8bit に量子化することで、ファイルサイズを削減できます。
*   **配布**: 変換後のファイル群（`model.json` と複数の `.bin` ファイル）は、プロジェクトの `public/models` ディレクトリに配置してください。

## 4. 学習データ（MIDI）の選定
「8ビットらしさ」を出すためのポイントです。

*   **MIDIの質**: ポケモンやレトロゲームのMIDIなど、シンプルな旋律と限られた同時発音数のデータを優先して学習させてください。
*   **データ拡張（Augmentation）**: ピッチを変更したデータを学習に含めることで、生成曲のバリエーションを増やせます。

## 5. アプリケーションへの組み込み
新しく作成したモデルを適用する手順：

1.  ファイルを `public/models/my-custom-model/` に配置。
2.  `src/lib/magentaService.ts` 内のモデル URL を新しく作成したパスに変更。
3.  `audioEngine.ts` で、新しいモデルが生成するノートの音色（Waveformなど）を調整。
